Google Sheets 設定指南

設定步驟

1

建立 Google Sheets 試算表

前往 Google Sheets 建立一個新的試算表，命名為「旅行分帳」或任何您喜歡的名稱。

重要：建立兩個工作表頁面：
  • Members - 用於儲存成員資料
  • Expenses - 用於儲存支出紀錄
2

設定試算表標題

在 Members 工作表的 A1 儲存格輸入「姓名」

在 Expenses 工作表的第一行輸入以下標題：

A1: ID
B1: 日期
C1: 項目
D1: 原始金額
E1: 台幣金額
F1: 幣別
G1: 付款人
H1: 分攤人員
I1: 狀態
3

取得試算表 ID

從瀏覽器網址列複製試算表 ID：

https://docs.google.com/spreadsheets/d/YOUR_SPREADSHEET_ID/edit

將這個 ID 記錄下來，稍後會用到。

4

建立 Google API 金鑰

  1. 前往 Google Cloud Console
  2. 建立新專案或選擇現有專案
  3. 在「APIs & Services」中啟用「Google Sheets API」
  4. 建立憑證 → API 金鑰
  5. 限制 API 金鑰只能使用 Google Sheets API（建議安全性設定）
5

設定試算表權限

在您的 Google Sheets 中，點擊右上角的「共用」按鈕，將試算表設定為「知道連結的任何人都可以檢視」或「知道連結的任何人都可以編輯」。

注意：這樣設定可能會讓其他人看到您的資料。如需更安全的設定，請研究 OAuth 2.0 認證方式。
6

更新應用程式設定

在 index.html 檔案中，找到 GOOGLE_SHEETS_CONFIG 區塊，並更新以下資訊： 

const GOOGLE_SHEETS_CONFIG = {
    API_KEY: '您的_API_金鑰',
    SPREADSHEET_ID: '您的_試算表_ID',
    MEMBERS_RANGE: 'Members!A:A',
    EXPENSES_RANGE: 'Expenses!A:I'
};

測試連線

完成設定後，重新載入應用程式。如果設定正確，您應該會在頁面右上角看到綠色的連線指示器，並收到「Google Sheets API 載入成功」的通知。

常見問題：
  • 如果 API 金鑰無效，會顯示紅色連線狀態
  • 如果試算表 ID 錯誤，資料載入會失敗
  • 如果工作表名稱不符，會顯示載入錯誤

功能說明

自動同步

每次新增、刪除成員或支出紀錄時，資料會自動同步到 Google Sheets

離線模式

如果無法連線到 Google Sheets，應用程式會自動切換到離線模式

資料備份

所有資料都會儲存在您的 Google Sheets 中，不會遺失
